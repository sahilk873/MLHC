# agents.md — MLHC 2026 Project Build Plan (BOLD + ENCoDE)

## Project one-liner
Build a fully reproducible pipeline that (1) quantifies pulse oximetry bias and hidden hypoxemia using BOLD at scale, (2) validates fairness conclusions using ENCoDE’s measured skin tone (not just race), and (3) proposes and tests debiasing/calibration strategies with rigorous external validation.

## Primary claim (what this paper contributes)
Race/ethnicity-stratified debiasing on retrospective ICU data can be misleading or incomplete. We will:
- Develop debiasing and calibration methods on BOLD (large, multi-center ICU EHR-derived).
- Externally validate on ENCoDE (prospective, skin tone measured across multiple devices and locations).
- Report whether improvements across race/ethnicity actually translate to improvements across measured skin tone (Monk scale, colorimetry/spectrophotometry metrics), plus site/device sensitivity.

## Research endpoints and definitions
We will compute these endpoints consistently in both datasets (as feasible):
1) Continuous error: e = SaO2 - SpO2
2) Absolute error: |e|
3) Hidden hypoxemia (primary): SaO2 < 88 and SpO2 >= T
   - Use sensitivity analysis over T in {90, 92, 94}. Default T=92 for main text.
4) Calibration: reliability curves for predicting SaO2 from SpO2 (and covariates)
5) Fairness metrics:
   - Gap in MAE across groups (race/ethnicity; and skin tone bins)
   - Gap in hidden hypoxemia FNR and FPR across groups
   - Worst-group performance (min-group AUROC or max-group FNR)
   - Uncertainty: bootstrap 95% CIs for all key metrics

Skin tone groups (ENCoDE):
- Primary: Monk Skin Tone Scale (administered visual scales) binned into 3–5 bins
- Secondary: continuous L* (CIE-L*) from spectrophotometer where available
- Also examine location and pulse-ox site sensitivity

# PHASE 4 — ENCoDE SaO2-SpO2 pair reconstruction (gold-standard logic)

## Goal
Reconstruct synchronized ABG (SaO2) and pulse-ox (SpO2) pairs in ENCoDE using time logic:
- SpO2 must be within 0–5 minutes before SaO2
- Choose nearest preceding SpO2 for each SaO2 within the same visit

## Tasks
1) Implement scripts/04_build_pairs_encode.py:
   - Identify SaO2 rows in MEASUREMENT using concept_id matching (Athena-mapped)
   - Identify SpO2 rows similarly
   - For each SaO2 measurement:
     - find candidate SpO2 with same person_id + visit_occurrence_id
     - SpO2_time <= SaO2_time
     - SaO2_time - SpO2_time <= 5 minutes
     - pick the max SpO2_time (nearest)
   - Output `artifacts/encode_pairs.parquet` (all pairs) with:
     - person_id, visit_occurrence_id
     - sao2_time, sao2
     - spo2_time, spo2
     - delta_minutes
     - plus joinable demographics (age, sex, race if present)
2) tests/test_encode_pairing.py:
   - Create a tiny synthetic MEASUREMENT fixture with known times
   - Assert pairing correctness and edge cases:
     - no preceding SpO2 -> dropped
     - multiple SpO2 candidates -> choose nearest
     - >5 minutes -> dropped

## Acceptance checks
- `python scripts/04_build_pairs_encode.py ...` creates encode_pairs.parquet (in `artifacts/`, all pairs)
- delta_minutes is within [0, 5] for all rows
- No duplicate (person_id, visit_occurrence_id, sao2_time) rows
- Tests pass

STOP if fewer than ~50 pairs are found (unless the dataset truly has fewer). Print counts and the matching rules used.

# PHASE 8 — Modeling and debiasing on BOLD

## Goal
Train correction models on BOLD, compare strategies, and prevent overfitting.

## Candidate models (start simple, then add complexity)
A) Monotone calibration curve:
- isotonic regression mapping SpO2 -> SaO2 (global)

B) Context-aware regression:
- Predict SaO2 from SpO2 + covariates (ridge regression, gradient boosting)

C) Group-aware calibration (sensitive feature included):
- Add race_ethnicity as a feature OR train group-specific calibrators

D) Fairness-regularized training (no race feature at inference):
- Train a model without race at inference but with fairness objective during training
- If too complex, replace with reweighting or group-robust optimization

## Tasks
1) Implement src/models/train.py:
   - Deterministic splits:
     - patient-level or hospitalization-level split (no leakage)
   - Cross-validation for hyperparameters
2) scripts/07_train_models.py:
   - Trains A–D on BOLD
   - Saves models to results/models/
   - Saves training logs and config snapshots to results/logs/
3) Unit tests:
   - Ensure split function prevents same patient across train/test where possible

## Acceptance checks
- Training completes end-to-end
- Model artifacts saved with versioned filenames
- Evaluation on BOLD test set is better than baseline (lower MAE) for at least models A or B
- No obvious leakage (verify unique IDs across splits)

STOP if model performance is worse than baseline across the board. Print feature availability and sanity-check SaO2/SpO2 types.
